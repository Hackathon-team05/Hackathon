2.3 シーケンス図

2.3.1 起動シーケンス（図6）

sequenceDiagram
  participant SW as スイッチ
  participant SRV as サーバーArduino
  participant INST as 楽器Arduino×4
  participant PC as Processing(PC)

  SW->>SRV: 電源ON
  SRV->>SRV: 自己初期化（PROGMEM展開・BPM設定・SPI初期化）

  loop CS送信（楽器1〜4順次）
    SRV->>INST: CSアサート（ハンドシェイク要求）
    alt 応答あり（2秒以内）
      INST-->>SRV: ハンドシェイク応答
      SRV->>SRV: アクティブ登録
    else タイムアウト（3回リトライ後）
      SRV->>SRV: スキップ登録（無音パート）
    end
  end

  SRV->>INST: BPMクロック一斉送信開始
  SRV->>PC: 音データシリアル送信開始
  SRV->>SRV: かえるの歌 自動演奏ループ開始

2.3.2 手動演奏シーケンス（図7）

sequenceDiagram
  participant PS as 圧力センサ
  participant INST as 楽器Arduino
  participant SRV as サーバーArduino
  participant PC as Processing

  PS->>INST: アナログ値 > 閾値（人形設置）
  INST->>INST: NoteOn検出・SoundData生成（velocity=127）
  INST->>SRV: SPI割り込み送信（SoundData 4byte, duration=255）
  SRV->>SRV: チェックサム検証OK・該当パート→手動モード（自動ミュート）
  SRV->>PC: シリアル転送（NoteOnフレーム type=0x01, SoundData 3byte）
  PC->>PC: パース → 加算合成 → ADSR → エフェクト → 音声出力

  Note over PS,PC: NoteOff

  PS->>INST: アナログ値 < 閾値（人形除去）
  INST->>INST: NoteOff検出（duration=0 SoundData生成）
  INST->>SRV: SPI割り込み送信（SoundData 4byte, duration=0）
  SRV->>SRV: duration=0検出・該当パート → 自動演奏モード復帰
  SRV->>PC: シリアル転送（NoteOffフレーム type=0x02, instrument_id 1byte）
  PC->>PC: Releaseフェーズ開始 → 減衰停止

2.4 Processing クラス構成（図8）

classDiagram
  class SerialParser {
    -Serial port
    -bytes buffer
    +void update()
    +SoundData parse(bytes raw)
  }
  class SoundData {
    +int instrument_id
    +int pitch
    +int velocity
  }
  class AdditiveSynth {
    -HarmonicTable[] tables
    -float sampleRate
    +float[] synthesize(SoundData sd, int n)
  }
  class HarmonicTable {
    +int instrument_id
    +float[] amplitudes
    +int[] orders
  }
  class Envelope {
    -float attack
    -float decay
    -float sustain
    -float release
    -int phase
    +float apply(float sample, boolean noteOn)
  }
  class Effects {
    +float vibrato(float s, float time, float depth, float rate)
    +float reverb(float s, float[] buf, int pos, float att)
  }
  class AudioOutput {
    -int bufferSize
    +void write(float[] samples)
    +void flush()
  }

  SerialParser --> SoundData : creates
  AdditiveSynth --> SoundData : reads
  AdditiveSynth --> HarmonicTable : references
  AudioOutput --> Effects : applies
  AudioOutput --> Envelope : applies
  AudioOutput --> AdditiveSynth : calls
  AudioOutput --> SerialParser : reads

2.5 物理データ設計（SoundData構造体）

フィールド名型バイト数値域説明
instrument_iduint8_t10〜3楽器番号（0:メロディ1, 1:メロディ2, 2:メロディ3, 3:リズム）
pitchuint8_t10〜127決定：MIDIノート番号（中央C=60）
velocityuint8_t10〜127音の強さ（手動演奏時は127固定）

▲ 表4: SoundData構造体 物理設計（合計3バイト固定長）。USBシリアルはNoteOn/NoteOff分離フレーム方式。NoteOff通知はtype=0x02フレームで独立送信（instrument_id 1byte）。SPI層ではduration=0のSoundData 4byteでNoteOff通知し，サーバーがUSBフレームへ変換する。

2.7 エラーハンドリング一覧

エラー種別検出条件対処復帰方法
SPI通信エラーチェックサム不一致が1楽器で3回連続該当楽器からのデータ受信停止・無音継続電源再投入（復帰なし）
ハンドシェイク失敗起動時タイムアウト2秒・3回後も無応答スキップ登録して演奏続行電源再投入
シリアル切断ProcessingがUSBシリアル切断検知バッファ内サンプル出力完了後に停止USB再接続
圧力センサ異常値ポーリング値がADCフルスケール付近閾値超過として扱い演奏継続自動回復
マイクノイズテンポ検出値がリングバッファ平均±2σ外れ外れ値として破棄、BPM更新なし自動回復
