# Arduinoシンセサイザー 仕様書

## 概要

サーバーArduinoのスイッチを入れると起動し、かえるの歌が演奏される。  
サーバーArduinoからクライアントのArduinoにSPI通信でクロックを同期させることでBPMを送り、割り込み処理で楽器ArduinoからサーバーArduinoへ音データが送られる。  
そのデータをサーバーArduinoにシリアル接続されているPC（Processing）にて解析し、各楽器ごとの音を正弦波の加算合成によって作成、エンベロープをかけたのち、ビブラートやリバーブといったエフェクトを付加して鳴らす。

## ハードウェア構成

### Arduino構成

| 種別 | 台数 | 役割 |
|------|------|------|
| サーバーArduino | 1台 | BPM管理・楽曲データ管理・PC通信・マイク入力 |
| 楽器Arduino（メロディ） | 3台 | 音データの送信 |
| 楽器Arduino（リズム） | 1台 | 音データの送信 |

### PC接続

- サーバーArduinoのみがPCとUSBシリアル通信で接続
- ProcessingがPCで動作し、音声を生成・出力する

### マイク

- サーバーArduino側に設置
- 種類：アナログマイクモジュール（例：MAX4466など、未確定）
- 用途：観客の拍手のテンポ検出

### 圧力センサ

- 楽器Arduino側に設置
- カエル人形を圧力センサの上に置くと、その楽器の音が鳴る

## 通信仕様

### サーバー ↔ 楽器Arduino（SPI通信）

- 一対多（1台のサーバーが4台の楽器Arduinoと通信）
- CSピン（チップセレクト）は楽器Arduino各台に1ピンずつ割り当て（**ピン構成は未確定**）
- サーバー → 楽器Arduino：BPMクロックを同期
- 楽器Arduino → サーバー：割り込み処理により音データを送信

### サーバー ↔ PC（USBシリアル通信）

- サーバーArduinoのみ接続
- 受信した音データをProcessingに転送

### 音データ構造体

```c
struct SoundData {
    uint8_t instrument_id;  // 楽器番号
    uint8_t pitch;          // 音の高さ（MIDIノート番号 or 周波数、未確定）
    uint8_t duration;       // 音の長さ
    uint8_t velocity;       // 音の強さ
};
```

> **未確定事項：** `pitch` をMIDIノート番号にするか周波数値にするか、各フィールドのバイト数・単位は要確定

### エラーハンドリング（チェックサム方式）

- 通信エラーが **1楽器につき3回** 発生した場合、その楽器からのデータ受信をフェイルセーフで停止する

## 楽曲データ管理

### 格納場所

- **サーバーArduinoのフラッシュメモリ（PROGMEM）に持たせる**

> **代替案：** 楽器Arduino側が自律的に演奏し、サーバーはBPMのみ管理する設計も可能。最終的な責務分担は要検討。

### 起動フロー

1. サーバーArduinoの電源ON（スイッチ投入）
2. サーバーArduino自己初期化

   - SPI・シリアル通信の初期化

4. PC側Processingが起動済みであることを確認（または確認なしで送信開始）
   - **TODO**：Processingとのハンドシェイクの有無を要決定
5. BPMクロックをアクティブな全楽器Arduinoへ一斉送信開始
6. かえるの歌の自動演奏を開始

## Processing（音声生成）

### 音声生成方式

- 正弦波の**加算合成**（各楽器の倍音構成を重ね合わせる）
- エンベロープ（Attack / Decay / Sustain / Release）を適用
- エフェクト：ビブラート、リバーブ

### 音色定義

- 実楽器のスペクトル解析結果をもとに倍音構成を決定する予定
- **現時点では未確定。プロトタイプ段階では正弦波単体で代替可能。**

### 受信〜発音フロー

Processing内での処理順序：

```
1. シリアルポートからバイト列を受信
2. SoundData構造体にパース（instrument_id / pitch / duration / velocity）
3. instrument_id から楽器の倍音構成テーブルを参照
4. 各倍音について正弦波を生成（周波数 = 基音 × 倍音次数、振幅 = 倍音係数 × velocity）
5. 全倍音の正弦波を加算し、1サンプルの波形値を合成
6. エンベロープ（ADSR）をサンプル単位で適用
   - Attack  : 音の立ち上がり（振幅 0 → max へ線形増加）
   - Decay   : ピーク後の減衰（max → sustain level）
   - Sustain : duration 中の持続レベル
   - Release : NoteOff 後の余韻（sustain level → 0）
7. エフェクト処理（適用順序は下記）
   a. ビブラート（周波数変調：低周波サイン波で音程を揺らす）
   b. リバーブ（遅延加算：ディレイバッファとアッテネーションで残響を付加）
8. 音声出力バッファへ書き込み → サウンドカード出力
```

> **未確定事項：** エフェクト処理順序の最終決定（ビブラート→リバーブ の順が一般的だが要確認）

### レイテンシについて

音声出力にはPCのサウンドバッファを介するため、**受信から発音までに遅延が生じる。**
BPMに合わせた音楽再生ではこの遅延が大きすぎると「ずれた演奏」になるため、バッファサイズの調整が重要になる。

| バッファ設定 | 特性 |
|------------|------|
| 大きい | 音切れしにくい・遅延が大きい（数百ms〜秒単位） |
| 小さい | 遅延が小さい・処理が追いつかないと音切れのリスク |
