# 機能分解構造（FBS: Function Breakdown Structure）
## Arduinoオーケストラ

---

## 構造図

```
【大項目】Arduinoオーケストラ
│
├── [中項目1] 起動・制御
│   ├── (小項目1.1) 起動シーケンス（自己初期化 → SPIハンドシェイク → 演奏開始）※TODO: タイムアウト設計
│   ├── (小項目1.2) BPM管理（デフォルトBPM設定・拍手テンポ同期）
│   ├── (小項目1.3) 楽曲データ管理（PROGMEM格納・演奏タイミング指示）
│   └── (小項目1.4) 演奏終了・停止処理（終了後の動作・強制停止）※TODO: ループ/停止 要決定
│
├── [中項目2] 通信
│   ├── (小項目2.1) SPI通信（BPMクロック送信・音データ受信・エラーハンドリング）※TODO: 復帰処理
│   └── (小項目2.2) USBシリアル通信（音データ転送）
│
├── [中項目3] センサ入力
│   ├── (小項目3.1) 圧力センサ処理（閾値判定・NoteOn/NoteOff・かえるの歌との共存）※TODO: 閾値・共存方式
│   └── (小項目3.2) マイク入力（拍手音検出 → BPM変更トリガ）
│
└── [中項目4] 音声生成
    ├── (小項目4.1) シリアル通信・パース（SoundData構造体への変換）
    ├── (小項目4.2) 加算合成（倍音構成テーブル参照・正弦波生成・加算）※倍音構成未確定
    ├── (小項目4.3) エンベロープ処理（Attack / Decay / Sustain / Release）
    ├── (小項目4.4) エフェクト処理（ビブラート → リバーブ の順で適用）
    └── (小項目4.5) 音声出力（バッファ書き込み・レイテンシ制御・停止時完了処理）
```

## 機能一覧（フラット表）

| 中項目 | 小項目 | 担当ノード | 備考 |
|--------|--------|-----------|------|
| 1. 起動・制御 | 1.1 起動シーケンス | サーバーArduino | 自己初期化 → SPIハンドシェイク → 演奏開始。TODO: タイムアウト・リトライ設計 |
| | 1.2 BPM管理 | サーバーArduino | デフォルトBPM設定＋拍手テンポ同期（ピーク検出・リングバッファノイズ除去） |
| | 1.3 楽曲データ管理 | サーバーArduino | PROGMEM格納・BPMクロックに合わせて演奏タイミング指示 |
| | 1.4 演奏終了・停止処理 | サーバーArduino | TODO: 終了後ループ/停止 要決定。強制停止時はクロック・SPI停止 |
| 2. 通信 | 2.1 SPI通信 | サーバーArduino ↔ 楽器Arduino | BPMクロック送信（1対多）・割り込み音データ受信・チェックサム（3回エラーでフェイルセーフ）。TODO: 復帰処理 |
| | 2.2 USBシリアル通信 | サーバーArduino ↔ PC | 音データをProcessingへ転送 |
| 3. センサ入力 | 3.1 圧力センサ処理 | 楽器Arduino | ポーリング・閾値判定（TODO: 実機調整）・NoteOn/NoteOff・かえるの歌との共存（TODO: 加算/割り込み） |
| | 3.2 マイク入力 | サーバーArduino | 拍手音ピーク検出 → BPM変更トリガ |
| 4. 音声生成 | 4.1 シリアル通信・パース | Processing | バイト列受信・SoundData（instrument_id/pitch/duration/velocity）パース |
| | 4.2 加算合成 | Processing | 倍音構成テーブル参照・正弦波生成・全倍音加算。倍音構成未確定 |
| | 4.3 エンベロープ処理 | Processing | Attack/Decay/Sustain/Releaseをサンプル単位で適用 |
| | 4.4 エフェクト処理 | Processing | ビブラート（音程変調）→ リバーブ（ディレイバッファ）の順 |
| | 4.5 音声出力 | Processing | バッファ書き込み・サイズ管理・レイテンシ制御・停止時バッファ完了処理。ライブラリ未確定（候補：Minim） |
