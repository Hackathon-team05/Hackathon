# SKILL: ハッカソン計画書作成支援

## 概要
ハッカソンの計画書をルーブリック（20点満点・5項目×4点）に沿って
Claudeが作成・添削・改善提案を行うためのスキル。

## トリガー条件
以下のような依頼でこのスキルを使う：
- 「計画書を書いて／直して／レビューして」
- 「ハッカソンの提出物を作りたい」
- 「計画書のどこを改善すればよいか教えて」

---

## ルーブリック（全5項目・各4点）

### 1. 背景と目的
| ランク | 基準 |
|--------|------|
| A (4点) | 技術的・社会的課題、既存システム、目的と独自性を参考文献つきで十分に記述 |
| B (2〜3点) | いずれかは記載されているが参考文献が不十分で根拠に乏しい |
| C (0〜1点) | 課題・目的・独自性のいずれも記載なし、参考文献なし |

**作成支援のポイント：**
- 解決したい社会課題・技術課題を具体的に述べる
- 既存の類似システムとの差別化ポイント（独自性）を明示する
- 主張の根拠となる論文・記事・統計データを参考文献として引用する

---

### 2. システムの全体像と構成要素
| ランク | 基準 |
|--------|------|
| A (4点) | 全体像と構成要素を図つきで十分に説明。計画書だけでシステムを構築できるレベル |
| B (2〜3点) | 図はあるが説明が不明瞭。計画書だけでは構築不可能 |
| C (0〜1点) | 全体像・構成要素のいずれも記載なし |

**作成支援のポイント：**
- システム構成図（アーキテクチャ図）を必ず含める
- 各コンポーネント（DB、API、UI等）の役割・連携方法を文章でも説明する
- 使用技術・ライブラリ・外部サービスを明記する
- 読者がゼロから再現できる粒度を目指す

---

### 3. 作業分担とスケジュール
| ランク | 基準 |
|--------|------|
| A (4点) | WBSで担当者が偏りなく割り当てられ、各タスクに適切なスケジュール |
| B (2〜3点) | WBS・スケジュールはあるが、偏りや無理な日程など部分的に不十分 |
| C (0〜1点) | 作業分担なし、スケジュールが適切でない |

**作成支援のポイント：**
- WBS（作業分解構成図）を表形式で作成し、タスク・担当者・期間を記載
- メンバー間の作業量が極端に偏らないよう確認する
- 開発・テスト・ドキュメント作成など全フェーズを網羅する
- バッファ（余裕日程）を設けた現実的なスケジュールにする

---

### 4. 図・表・フローチャート
| ランク | 基準 |
|--------|------|
| A (4点) | 構成図・フローチャート・WBS・スケジュール表が適切に記載され、文中で説明されている |
| B (2〜3点) | 図表は揃っているが一部不明瞭で説明が不十分 |
| C (0〜1点) | 必要な図表のいずれかが欠けており、説明も不十分 |

**作成支援のポイント：**
- 必須図表チェックリスト：
  - [ ] システム構成図
  - [ ] 動作フローチャート
  - [ ] WBS（作業分解構成図）
  - [ ] スケジュール表（ガントチャート推奨）
- 図表には通し番号とキャプションをつける
- 本文中で「図1に示すように〜」と必ず言及する

---

### 5. 評価指標と数値目標
| ランク | 基準 |
|--------|------|
| A (4点) | 目的達成の可否を判断する評価指標と具体的な数値目標が適切に設定・記載されている |
| B (2〜3点) | 評価指標・数値目標は記載されているが適切でない |
| C (0〜1点) | 評価指標・数値目標が記載されていない |

**作成支援のポイント：**
- 「何を・どの数値で・どれくらい達成したら成功か」を明記する
- 例：「認識精度90%以上」「応答時間1秒以内」「ユーザー満足度4.0/5.0以上」
- 目的と直結した指標を選ぶ（目的が「省力化」なら「作業時間削減率」など）
- 測定方法・測定タイミングも併記するとなおよい

---

## Claudeの行動指針

1. **新規作成**：ユーザーからシステムの概要を聞き出し、上記5項目すべてを満たす計画書の草案を生成する。不足情報はヒアリングで補う。
2. **添削・レビュー**：提出された計画書を5項目ごとにランク判定し、改善点を具体的に提示する。
3. **部分改善**：特定セクションのみ改善を求められた場合は、そのセクションのルーブリックに集中して対応する。
4. 図表はMarkdownの表やMermaid記法で代替し、ユーザーに清書を促す。
